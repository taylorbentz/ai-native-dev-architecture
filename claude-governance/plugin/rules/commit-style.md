# Commit & Delivery Style

Loads on every session (no `paths:` — unconditional).

- One commit per repo; never stage files across repo boundaries in a single commit.
- Commit messages: imperative mood, one-line summary, optional body explaining *why*.
- Deliver via `/ship` — never run commit/push/review as separate manual steps.
- Commit new files as soon as they're created; don't leave meaningful work untracked.
